## 1.2 计算机系统的组成

- **硬件是什么**：硬件指的是有形的物理设备，是计算机系统中实际物理装置的总称。

- **软件是什么**：软件是指在硬件上运行的程序和相关的数据以及文档。
  
  > 对于某一功能来说既可以用硬件实现有可能用户软件实现，则称软硬件在逻辑功能上等价的。

- **冯·诺依曼机四大特点**：
  
  1. 采用存储程序的工作方式
  
  2. 计算机硬件系统由**运算器、存储器、控制器、输入设备、输出设备**五大部件组成
  
  3. 指令和数据以**同等地位**存储在存储器中，形式上没有区别么，但计算机应该能区分它们
  
  4. 指令和数据**都是二进制**代码来表示。指令由操作系统和地址码组成，操作码指出操作类型，地址码指出操作数的地址
     
     > 对于第三点，我们可以通过当前的流程来判断是指令还是数据。 

- 存储程序的基本思想：将事先编制好的程序和原始数据送入主存才能执行，一旦程序被启动执行，就无须人员的操作干扰，计算机会逐条自动执行指令，直到程序执行结束

- <mark>地址寄存器MAR（Memory Address Register)与数据寄存器MAR（Memory Data Register）</mark>：MAR存放**访问存储**的地址，经过地址译码过后找到所选的存储单元。MDR用于暂存要从存储器中读或写的信息。MAR用于寻址，其位数对应着存储单元的格式，如MAR为10位，那则有$2^{10}=1024$ 存储单，记为**1k**，MAR的长度和<mark>程序计数器PC（Program Counter）</mark>的长度是相等的。MDR的位数和存储字长想等，一般为字节的二次幂整倍
  
  > MAR和MDR虽然是存储器的一部分，但在现代计算机中确实存在于CPU中的，另外，高速缓存（Cache）也存在于CPU当中

- <mark>控制器由PC和、指令寄存器IR(Instruction Register)和控制单元CU（Controller Unit）</mark>组成，PC用来存储当前欲执行指令的地址，具备自动加一的功能，可以自动形成下一条指令的地址，它与贮存的MAR有一条直接通道。IR是用来存放当前的指令，其内容来自于主存的MDR，指令中的操作码OP（IR）送至CU，用来分析指令并且发出各种微操作命令序列；而地址码Ad（IR）则送往MAR，用来取操作数。
  
  > 这里的加一指的是一条指令的长度。

- 系统和软件与应用软件：系统软件如OS或者DBMS 等，而应用软件如QQ什么的都算。

- 三个级别的语言：
  
  1. 机器语言：又称二进制语言，是**唯一**可以识别和执行的语言。
  
  2. 汇编语言：汇编语言就是用英文单词或者缩写来代替二进制的指令代码。汇编语言程序必须要经过翻译转换成机器语言之后，才能在计算机的硬件系统上执行。
  
  3. 高级语言：如Java、C++等这样的方便程序猿编写代码的程序。**高级语言需要经过编译程序编译成汇编语言，然后再由汇编操作得到机器语言，或者直接又高级语言翻译成机器语言程序**

- 翻译程序有以下三类：
  
  1. 汇编程序：将汇编语言程序翻译成机器语言程序
  
  2. 解释程序：将源程序中的语句按照顺序逐条翻译成机器指令并且立即执行
     
     > 边翻译成机器指令，一边执行
  
  3. 编译程序：将高级语言程序翻译成汇编语言或者直接翻译成机器语言程序。

- 计算机系统层次结构的分层方式目前尚无统一标准，常见的一种见图。对于某层的观察者而言，只需要关注该层的语言和了解来使用计算机，而不必关心下层是如何工作的。**层次与层次之间联系紧密，下层是上层的基础，上层是下层的拓展**

- 存储程序工作方式：程序执行前，需要将程序所包含的指令和数据送入主存，一旦开始执行，就不需要操作人的干预，自动逐条的完成指令的取出和执行任务。一个程序的执行就是周而复始地执行一条一条指令的过程。每条指令的操作过程包括：从主存取指令=>对指令开始译码=>计算下条指令地址=>取操作数并执行=>将结果返回给存储器。
  
  1. 预处理阶段：预处理器将=队员程序中以字符#开头的命令进行处理。
  
  2. 编译阶段：编译器对预处理后的源程序进行编译，生成一个汇编语言程序
     
     > 汇编语言中的每一条语句都以一种文本格式描述了一条低级机器语言指令。
  
  3. 汇编阶段：汇编是将编译阶段生成的文件翻译成机器语言指令，并且包成一个可恶重定位目标的二进制文件。
  
  4. 链接阶段：链接器将多个可重定位目标文件和标准库函数合并成为一个可执行文件。最终这个可执行文件被保存在磁盘上。

- <mark>计算机的主要性能指标</mark>：
  
  1. 字长：自长治的是计算机经过一次整数运算（定点整数运算）所能处理的二进制数据的位数，通常与CPU的寄存器位数、加法器有关。因此，字长一半等于内部寄存器的大小，子长越长，书的表示范围越大，计算精度越高。计算机自唱通常定位字节（8位）的整数倍。
  
  2. 数据通路带宽：指的是数据总线一次性所能并行传送信息的位数。这里指的是外部数据总线的宽度，它和CPU内部的数据总线宽度（内部寄存器的大小）有可能有不一样
  
  3. 主存容量：主存容量指的是主存储器能存储信息的最大容量，一般是以字节来衡量，也可以用$ 字数*字长$来表示存储容量。MAR得位数翻译了存储单元的个数，MDR的位数反应了存储单元的字长。例如，MAR为16位，则 $2^{16}=25536 $，即此存储单元内有65536个存储单元（可称为64K内存，$1K = 1024 $）若MDR为32位，表叔存储容量为$ 64K*32位$ 。
  
  4. 运算速度：
     
     - 吞吐量：指的是系统在单位时间内处理请求的数量。它就取决于信息能多快的输入内存，CPU能多快的取指令，数据能多快的从内存取出或存入，以及所得结果能多快的从内存送给一台外部设备。几乎每部都关系到主寸，因此系统吞吐量主要取决于**主存的存取周期**。
     
     - 响应时间：指 从用户向计算机发送一个请求，到系统对该请求作出响应并获得所需结果的等待时间。一般包括CPU时间（运行一个程序所花的时间）与等待时间（用于磁盘访问、存储器访问、I/O操作、操作系统开销等时间）。
     
     - <mark>CPU时钟周期</mark>：通常为节拍脉冲或者T周期，即主频的倒数，它是CPU中的最小时间单位，执行指令每个动作至少需要1个时钟周期。
     
     - <mark>CPU时钟频率</mark> :也叫主频，是机器内部主石重的频率，是衡量机器速度的重要参数。对于同一个型号的计算机，其主频越高，完成指令的一个步骤所需要的时间越短，执行指令速度越快。
       
       > CPU时钟周期 = 1/主频，主频通常以Hz为单位，1Hz 表示每秒一次。
     
     - CPI：即执行一条指令需要的时钟周期数。不同指令的时钟周期数可能不一样的，因此对一台机器来说，CPI一般指的是改哦程序或者该机器指令集中的所有指令执行所需要的**平均时钟周期数** 
     
     - CPU执行时间：指运行一个程序所花费的时间。$CPU执行时间 = CPU始终周期数/主频 = （指令条数*CPI）/主频 $ 这个公式表示CPI和指令条数是互相制约的。例如，更该指令集可以减少程序所含的指令数，但同时可能引起CPU结构的调整，从而可能会**增加时钟周期宽度也就是降低主频** 
     
     - MIPS：即每条执行多少百万条指令。 $MIPS = 指令条数/(执行时间*10^6)= 主频/(CPI*10^6) $ MFLOPS（每条执行多少百万体次浮点运算）$MFLOPS = 浮点操作数/(执行时间*10^6)$、GFLOPS、TFLOPS、PFLOPS、EFLOPS、ZFLOPS。都是类似的公式，区别在于次方多少。
